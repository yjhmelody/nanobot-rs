# RFC-0004: 记忆系统与子代理升级

## 元信息

| 字段 | 值 |
| --- | --- |
| 文档状态 | `Proposed` |
| 实施状态 | `Not Started` |
| 提出顺序 | `4` |
| 优先级 | `P2` |
| 依赖 | `RFC-0001` |
| 最后更新 | `2026-04-01` |

## 摘要

把 `nanobot-rs` 的 session、memory 和 spawn 能力升级成更稳定的高层工作模式：记忆分层、检索更智能、后台提炼更受控，子代理可观察、可恢复、可限额。

## 当前不足

### 记忆系统过于粗粒度

- 长期记忆读取没有结合用户 query
- 写入方式偏覆盖式，难以积累结构化经验
- 没有区分 session 临时记忆、项目长期记忆、用户手工记忆

### 会话智能不足

- consolidation 有基础能力，但策略暴露不足
- 缺少“何时该压缩、何时该提炼、何时该写入长期记忆”的高层策略

### 子代理仍偏底层执行

- 子代理创建后缺少统一状态查询
- 缺少预算、超时、工具边界、结果归并机制
- 后台工作项没有统一列表和恢复能力

## 目标

- 在不引入复杂外部存储的前提下，把本地优先能力升级成更聪明的记忆层
- 让子代理既能被利用，又不会失控
- 降低长上下文任务的 token 压力

## 方案

## 1. 分层记忆模型

建议拆为三层：

### Session Memory

- 与当前会话强相关
- 生命周期短
- 主要用于多轮连续任务

### Project Memory

- 面向当前 workspace
- 存储稳定事实、约定、常见命令、架构知识
- 由后台提炼流程维护

### Manual Memory

- 用户显式写入或确认的重要规则
- 优先级高于自动提炼内容

## 2. Query-aware 检索

升级 `get_context`：

- 根据 query 选择相关记忆块
- 支持按标签、来源、最近使用时间、重要度排序
- 控制回填 token 上限

最小版本不引入向量库，先做：

- 关键词索引
- tag + metadata 检索
- 简单评分排序

## 3. 背景记忆提炼 Worker

新增受限背景任务：

- 输入：最近对话、工具结果、用户反馈
- 输出：候选 session summary、候选 project memory 更新
- 约束：只允许少量只读工具，不允许危险 shell 或写文件

建议新增：

- `MemoryWorker`
- `MemoryExtractionJob`
- `MemoryCandidate`

## 4. 会话压缩与记忆写入联动

将 consolidation 从“消息达到阈值就压缩”升级为更明确策略：

- 触发条件
- 压缩级别
- 是否抽取长期记忆
- 是否保留关键工具结果

## 5. 子代理生命周期管理

新增统一子代理任务模型：

- `Queued`
- `Running`
- `WaitingApproval`
- `Completed`
- `Failed`
- `Cancelled`

同时记录：

- 发起 session
- 任务目标
- 可用工具范围
- 模型 / 预算 / 超时
- 结果摘要

## 6. 子代理预算与隔离

建议支持：

- 最大 token
- 最大工具调用次数
- 可用工具 allowlist
- 可选独立工作目录或工作区子目录

## 7. 结果归集

子代理完成后产出结构化结果：

- 摘要
- 关键发现
- 工具执行统计
- 产出文件列表
- 是否需要人工审阅

## 实施阶段

| 阶段 | 范围 | 状态 | 说明 |
| --- | --- | --- | --- |
| 阶段 1 | 记忆配置显式化、query-aware 检索最小版、子代理状态查询最小版 | `Planned` | 先把能力从“存在”变成“可用” |
| 阶段 2 | 背景记忆提炼 worker、project/manual memory 分层、子代理预算与 allowlist | `Planned` | 增强长期稳定性 |
| 阶段 3 | 会话恢复时主动注入相关记忆、结果结构化归档、任务面板联动 | `Planned` | 打通完整工作流 |

## 风险与取舍

- 记忆抽取太激进会写入噪声
- 子代理如果没有强预算和工具边界，排障会非常困难
- 不建议一开始引入重型向量数据库，先做好本地文件和轻量索引

## 验收标准

- 记忆检索会根据 query 返回相关内容，而不是全量硬塞
- 至少一种长期记忆可以自动提炼并被后续会话使用
- 子代理可被列出、查询状态、查看结果摘要
- 子代理预算和工具边界可配置
